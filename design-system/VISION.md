# 日记库改造 — 理想目标

> Outer Wilds 星际拓荒美学 · 从归档工具到个人档案馆的沉浸式改造
> 最终修订：2026-06-04

---

## 零、总体体验弧线

用户打开日记库，直接置身于飞船驾驶舱中。窗外是运转中的太阳系——6 颗行星沿轨道环绕中央太阳。用户通过 Shift 在"窗景观赏"和"操控台操作"之间切换。选择维度 → 摄像机自动飞向目标行星 → 锁定旋转观察 → 点击地表区域 → 折跃传送进入行星内部。

全程在一个连续的太空叙事里。不出现传统列表/表格。不跳页面。

```
驾驶舱（操控台模式）  ⇄Shift⇄  驾驶舱（窗景模式）
       ↓ 选 tag
  摄像机曲线飞向行星
       ↓ 锁定旋转
  点击球面区域
       ↓ 折跃门
  行星 3D 地表漫步
       ↓ 点击热键元素
  日记阅读浮层
```

## 一、场景架构

### 1.1 双模式系统

用户始终在驾驶舱内。驾驶舱由视频素材 `cockpit_idle_frame.png` 作为全屏底图（抠绿后替换舷窗为 Three.js 渲染的星空）。Shift 键在两种模式间切换：

| 模式 | 看到什么 | 能做什么 |
|------|----------|----------|
| **操控台模式**（默认） | 驾驶舱全貌——窗框、控制台面板、示波器屏幕、仪表盘、按钮。窗外星空可见于上方舷窗。 | 操作示波器切换 tag、点击确定按钮、唤出侧栏检索。 |
| **窗景模式** | 干净的全屏星空视野，无驾驶舱叠加。屏幕角落显示极简按键提示 UI。 | 拖拽旋转视角观察太阳系、看到行星轨道实时运转。 |

**切换规则：**
- 初始默认：操控台模式，窗外太阳系全景
- 选择行星 tag 并确定 → 自动切换到窗景模式 → 摄像机飞向行星
- 锁定行星后，操控台面板逐渐淡入，回到操控台模式
- 用户可随时按 Shift 手动切换

### 1.2 太阳系初始态

窗外是 Three.js 渲染的 3D 太阳系场景。6 颗维度行星（情感/生活/实践/创作/成长/时间）沿各自的椭圆轨道绕中央太阳运行。每颗行星有不同的颜色和纹理暗示。

太阳：暖橙色发光球体，直径约为屏幕的 1/8。不带日冕特效。不是交互热键。

行星轨道：以微妙的不同倾角交叉，轨道线极细（1px），半透明米白/淡金色。

背景：三层星场粒子。远景静止、中景慢速、近景较快。

### 1.3 操控台（视频贴图叠加层）

操控台模式使用四段视频的关键帧素材叠加在 3D 场景之上：

| 素材 | 作用 | 处理 |
|------|------|------|
| `cockpit_idle_frame.png` | 驾驶舱全貌底图 | 舷窗区域抠绿透明，露出 Three.js 星空 |
| `detector_idle_frame.png` | 左侧示波器/探测器面板 | 屏幕区域叠 HTML/CSS 绿色 CRT 文字 |
| `helmet_idle_frame.png` | HUD 四角暗角 + 环形仪表 | 压暗画面边缘，强化沉浸感 |

### 1.4 示波器 Tag 切换器

位于操控台中央或左侧的 CRT 屏幕区域。HTML/CSS 覆盖在探测器/仪表屏幕位置。

**显示格式（等宽荧光绿，Space Mono）：**

```
┌─────────────────────────┐
│  ～～～～～～～～～～  │  ← 波形线（装饰，受维度日记数据驱动）
│                         │
│    <   情  感   >       │  ← 当前维度名，大字居中
│                         │
│    4 BRANCHES           │  ← 子分类数量
│    175 LOGS             │  ← 该维度日记总数
└─────────────────────────┘
```

**交互方式：**
- 点击 `<` 或 `>` 尖括号 → 维度在 6 个之间循环切换（情感↔生活↔实践↔创作↔成长↔时间↔太阳系）
- 字符切换时：旧文字逐行扫描线消失（从上到下）、新文字打字机效果逐字出现
- 尖括号点击时有短暂的绿色增亮反馈
- 屏幕有水平扫描线纹理叠加

**确定按钮：** 操控台上的一个物理按钮位置（视频素材中的某个按钮），点击 → 触发摄像机飞行动画。

**初始 tag 为"太阳系"：** 窗外显示 6 颗维度行星。切换到某个维度 tag 并确定 → 窗外行星切换为该维度的子分类行星。切换回"太阳系"→ 恢复 6 颗维度行星。

### 1.5 侧栏检索面板

中键唤出圆形光标（淡橙外圈+十字准星）→ 移动到屏幕左边缘 → 侧栏从左侧展开。

**侧栏外观：** 飞船仪表抽屉式面板——深棕底色 + 橙色细线边框 + Space Mono 数据文字。280px 宽，从屏幕顶部延伸到底部。

**侧栏内容：**
- 顶部：搜索输入框，暗底+橙色边框
- 中部：按日期排列的日记列表（日期等宽+标题衬线+正文片段）
- 底部：条目计数
- 点击某条 → 如果已有行星锁定则飞向该日记位置；如果无则先切换到对应维度

## 二、摄像机系统

### 2.1 太阳系全景（初始位置）

摄像机位于太阳系上方偏前的固定位置。视野覆盖整个行星系统。所有 6 颗行星可见。此状态下：
- 拖拽鼠标 → 以太阳为中心旋转视角（类似 orbit controls，但有一定范围限制）
- 摄像机不会远离到看不到行星的程度
- 滚轮 → 小幅缩放（但不能飞入行星，只是调整观察距离）

### 2.2 自动飞行（选 tag 确定后）

用户选了维度 tag 并点确定后：

1. 目标行星高亮（增亮+发光环扩张）。
2. 摄像机从当前位置沿一条平滑的三次贝塞尔曲线飞向目标行星。
3. 飞行曲线：先略微后退/上升（给"出发"的感觉），然后加速飞向行星，最后减速靠近。
4. 飞行时长约 1.2-1.8 秒（取决于距离）。
5. 飞行过程中，窗外其他行星仍在轨道上运行——能看到整个太阳系的实时动态。
6. 到达后，摄像机锁定在距行星表面一定距离的位置。

### 2.3 锁定行星旋转（到达后）

摄像机锁定在行星的球心坐标系上。行星直径占屏幕竖向高度约 40%。用户可以：
- 拖拽鼠标 → 围绕行星旋转（经度+纬度限制，不能翻到底部）
- 滚轮 → 无缩放（距离固定，锁定相对静止）
- 行星表面可见 2-4 个颜色/纹理明显不同的区域（子 tag 分区）

行星在太空中仍然沿轨道运行，但摄像机与之保持相对静止。背景中可以看到其他天体缓慢移动——给用户"我在这里，但宇宙还在运转"的感觉。

操控台面板逐渐淡入，回到操控台模式。

### 2.4 返回太阳系

用户通过操控台上的"离开行星"按钮或侧栏中的返回选项 → 摄像机从行星位置飞回太阳系全景的初始位置。飞行曲线反之：先后退，然后弧线上升回到俯瞰视角。时长约 1-1.5 秒。

## 三、行星球面区域

### 3.1 区域划分

锁定行星后，行星表面的 2-4 块区域以不同颜色/纹理区分。区域采用球面等跨划分（基于子 tag 数量）。例如：

- 4 个子 tag → 行星表面分为 4 个经度象限区带（类似橙子瓣）
- 3 个子 tag → 3 个 120° 区域
- 2 个子 tag → 两个半球

区域之间的边界是柔和的渐变过渡，不是硬边。

### 3.2 区域识别与选择

- 每个区域有一个微妙的光晕或描边暗示"可选"。
- 鼠标悬停某区域 → 该区域微微增亮 + 区域上方浮现子 tag 名称标签（Space Mono 小字，淡绿荧光）。
- 区域高亮时，周围出现一个圆形或菱形的瞄准框（HUD 风格，橙色细线）环绕该区域。
- 点击 → 触发折跃门转场。

### 3.3 不同维度的行星外观

| 维度 | 行星主色调 | 区域区分方式 | 子 tag 示例 |
|------|-----------|-------------|------------|
| 情感 | 柔和暖色到冷色渐变 | 色温区分——暖橙→冷蓝的球面渐变，每瓣不同色温 | 平静(淡蓝)、低落(深蓝紫)、焦虑(暗红棕)、渴望(暖金橙) |
| 生活 | 棕色大地色调 | 地貌纹理区分——有的区域平滑(校园)、有的粗糙(出行) | 校园(灰白)、职场(金属灰)、出行(蓝)、日常(暖棕) |
| 实践 | 青灰色 | 纹理密度区分——有的区域密集纹理(创作)、有的稀疏(身体) | 身体(暖红棕)、创作(青灰)、阅读(米黄) |
| 创作 | 深色基底+彩色光斑 | 光斑颜色区分——金色光斑(文字)、蓝色光斑(影像)、白/紫光斑(内化) | 文字(墨绿)、影像(深蓝)、内化(半透明紫) |
| 成长 | 过渡渐变 | 纹理复杂度区分——简单(自省)、中等(关系)、复杂(转折) | 自省(银灰)、关系(暖粉金)、转折(双色分界) |
| 时间 | 带轨道环的暗色球体 | 表面按年代纹理——稀疏(大一)、中等(大二上)、密集(大二下) | 每个学期=一个区域 |

## 四、折跃门转场系统

### 4.1 视频素材

使用视频素材"进入/退出.mp4"导出的 `portal_enter.webm`。

素材内容：从干净画面 → 菱形折跃门打开/展开 → 画面逐渐收缩 → 完全黑屏。类似"闭眼"的效果。

### 4.2 进入行星流程

```
窗景（锁定行星，选中区域）
  → 点击区域
  → portal_enter.webm 正放（~1.5s，画面收缩至黑）
  → 全黑（保持 0.3-0.5s，后台加载行星地表 3D 场景）
  → portal_enter.webm 倒放（~1.5s，画面从黑展开）
  → 行星 3D 地表场景呈现
  → 操控台面板缓慢淡入（~1s）
```

倒放等价于"睁眼"——从黑屏中打开，看到的已经是新场景。

### 4.3 退出行星流程

```
行星地表
  → 点击"离开行星"或侧栏返回按钮
  → portal_enter.webm 正放（画面收缩至黑）
  → 全黑（后台切换回太阳系场景）
  → portal_enter.webm 倒放（画面从黑展开）
  → 回到太阳系窗景（摄像机在之前锁定的行星位置）
  → 操控台面板淡入
```

### 4.4 跨视频素材转场

如果后续有进入不同行星类型的特殊转场（如时间维度进入轨道环而非地表），可用 portal 视频作为统一的基础转场，在倒放前通过场景切换改变最终画面。

## 五、行星内部：3D 地表漫步

（以下保持之前的设计，在折跃门倒放后呈现）

### 5.1 进入行星

折跃门倒放完成 → 镜头落在行星地表上，面对前方广阔的异星景观。

### 5.2 空间感知：小星球 + 鱼眼曲率

用户站在一颗直径非常小的星球上。因为星球太小，地平线向上弯曲成明显弧形。屏幕左右边缘的地表向上弯起，像透过鱼眼镜头看世界。天空颜色因维度而异，但始终保持暗调。

### 5.3 地表热键元素

地面上散布着小物件——石头、花草、虫、晶体、雕像、碎片——作为日记的热键入口。低多边形风格（与 Outer Wilds 一致），屏幕空间约 20-60px 高。

日记物体有微弱的发光高亮边缘（暖橙色细线），非日记物体无发光纯装饰。

### 5.4 移步换景：滚轮前进

鼠标滚轮驱动视角沿预设轨道向前平移。非 WASD 自由移动。一镜到底的连续前进。新的元素从前方地平线升起，身后的元素逐渐远去消失。行星是球形的，前进足够久会环绕一圈回到起点。

### 5.5 日记交互

- 鼠标悬停日记物体 → 物体微微浮起 + 增亮 + 标题气泡浮现
- 点击 → 视角平滑拉近 → 物体展开为 A4 纸张风格阅读面板
- 面板有"纸质"纹理叠加 + 柔和阴影 + 飞船仪表橙色细线框
- 关闭（Escape 或点击面板外）→ 面板缩小回物体尺寸 → 视角回位

### 5.6 子 tag 切换（行星内部）

进入行星后，下设的小 tag 通过操控台模式的示波器切换（和维度切换共用同一个 UI 逻辑）。不再用实体行星表示子 tag。切换子 tag 时：
- 地表生态渐变切换（同一行星的不同区域地貌）
- 热键元素类型随之变化
- 示波器显示当前子 tag 名称

### 5.7 不同维度行星的氛围差异

| 维度 | 天空 | 地面 | 光照 | 氛围 |
|------|------|------|------|------|
| 情感 | 深蓝到紫渐变，慢速流动的极光带 | 砂砾地，偶尔有浅水洼有微光倒影 | 柔和的漫射光，没有明确光源方向 | 安静，沉思，像深夜的海边 |
| 生活 | 暖棕暮色，远处有微弱的"城市灯光"在地平线下 | 松软的泥土或细沙 | 暖色调的侧面光，有长阴影 | 熟悉，亲切，像黄昏时的旧街区 |
| 实践 | 青灰天空，有快速移动的薄云 | 硬质地面，有工具的刻痕和印迹 | 清晰的顶光，阴影锐利 | 工作坊，专注感 |
| 创作 | 深色但有彩色光斑漂浮（像颜料在水中扩散） | 灰白画布般的平整地表 | 多方向的柔光，物体阴影不统一 | 艺术工作室，灵感碎片散落 |
| 成长 | 从暗→亮的渐变天空，像黎明前 | 有裂缝和断层的地表，但裂缝中长出细小植被 | 侧逆光，物体边缘有轮廓光 | 过渡期，破晓感，不确定但充满可能 |

## 六、时间维度的特殊处理

时间是唯一的"功能型"维度。不进入 3D 地表——而是进入**多层轨道环系统**。

### 6.1 轨道环空间

折跃门倒放后，用户悬浮在空间中。面前是由 3 条同心椭圆轨道环组成的立体结构。每条环 = 一个学期的时间线。

环上分布小光点 = 日记。光点按日期排列，颜色表示情感基调（暖=积极/渴望，冷=低落/焦虑），大小表示日记长度。

3 条轨道环以不同角度倾斜交叉（大一环下方、大二上环中间、大二下环上方），形成三层结构。

### 6.2 交互

- 悬停光点 → 光点变大增亮 + 日期标题浮现
- 点击 → 打开日记阅读浮层
- 滚轮 → 沿环滑移视角
- 侧栏 → 标准检索 + 按月分组索引

## 九、动效体系

### 9.1 时序节奏

| 类型 | 时长 | easing | 用途 |
|------|------|--------|------|
| 微型交互 | 150-200ms | ease-out | 按钮按下、悬停增亮、尖括号点击反馈 |
| 标准过渡 | 300-500ms | ease-in-out | 侧栏展开/收起、Shift 模式切换、标题气泡浮现 |
| 摄像机飞行 | 1000-1800ms | 自定义缓动（三次贝塞尔） | 飞向行星、返回太阳系 |
| 折跃门正/倒放 | 各 ~1500ms | 视频帧自然速度 | 进出行星转场 |
| 粒子聚合 | 2000-3500ms | 模拟物理阻尼 | 维度切换时窗外星点重排 |
| 环境呼吸 | 3000-6000ms | 正弦波式 | 太阳脉动、指示灯呼吸 |

### 9.2 运动语言

- 摄像机飞行曲线：先略微后退/上升 → 加速 → 减速靠近 → 停稳。贝塞尔控制点可调。
- 粒子运动遵循简化 Boids 规则：分离+对齐+凝聚。
- 所有 UI 过渡使用 `transform` 和 `opacity`，不触发 layout/paint。

### 9.3 性能与无障碍

- `prefers-reduced-motion` 时降级为静态或极简过渡
- 粒子总数：太阳系星场 ~200，窗外近景 ~150
- 3D 地表使用 instanced mesh 优化同类物件
- 视频素材预加载，避免折跃门播放时卡顿
- 场景切换时后台异步加载，黑屏时长即加载时间

## 十、技术实现

### 10.1 技术选型

| 层次 | 方案 | 备注 |
|------|------|------|
| 3D 场景（太阳系+行星） | Three.js + React Three Fiber | 球体+轨道+摄像机+raycaster |
| 驾驶舱贴图叠加 | HTML/CSS img 层在 Three.js canvas 之上 | 固定全屏 PNG，舷窗区域透明 |
| 示波器 CRT | HTML/CSS 层覆盖在探测器屏幕位置 | 绿色荧光文字+扫描线+打字机动画 |
| 粒子系统 | Three.js Points 或 Canvas 2D | 星场、聚合/散开动画 |
| 折跃门转场 | HTML5 Video 元素全屏叠加 | 预加载 WebM，正放+倒放切换 |
| 行星地表 3D | R3F + Drei | 小型球体 + 低模物件 + 轨道摄像机 |
| UI 覆盖层 | React 组件 + framer-motion | 侧栏、日记浮层、HUD、按键提示 |

### 10.2 关键交互实现

**Shift 双模式切换：**
- 状态变量 `viewMode: 'cockpit' | 'window'`
- cockpit 模式：叠加驾驶舱 PNG + 示波器 HTML + HUD
- window 模式：仅 Three.js canvas + 极简按键提示
- 切换动画：叠加层 opacity 过渡 300ms

**摄像机飞行：**
- 用 GSAP 或 Three.js TWEEN 沿贝塞尔曲线插值
- 目标位置和方向由目标行星的 world position + 偏移量计算
- 飞行中用户不可拖拽（防止冲突）

**行星球面区域检测：**
- 鼠标在行星上的 UV 坐标 → 判断落在哪个经度区间
- raycaster 检测鼠标是否命中行星 mesh
- 悬停反馈：调整对应区域的 material emissive

**折跃门正放+倒放：**
- `<video>` 元素全屏 absolute 定位，z-index 最高
- 正放：video.currentTime 正向播放到结尾（黑屏）
- 倒放：video.currentTime 从结尾逆向播放到开头
- 或预先导出正放和倒放两个 WebM，避免 JS 倒放性能问题

### 10.3 数据流

所有数据仍从 server.js API 读取。日记元数据（tags、日期、分类映射）在客户端从已有 markdown frontmatter 计算。维度/子分类的日记分配在客户端维护一个映射表。
