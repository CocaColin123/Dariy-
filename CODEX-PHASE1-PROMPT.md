## 第一件事：输出你的理解计划

用你自己的话告诉我：

1. **你理解的整个体验流**——从打开应用到读完一篇日记，用户经历了什么？
2. **Phase 1 你要建什么**——拆成哪几个模块，每个模块的职责和技术方案
3. **你先做哪个模块**——为什么那个先做（依赖最少？最核心？最容易验证？）
4. **你预计的可能卡点**——技术上的不确定项或需要我确认的地方

格式随意，不要套模板。我读完后我们讨论，然后你开工。

---

## 第二件事：然后开始 Phase 1

Phase 1 的目标是：**打开应用能看到太阳系在旋转，Shift 能切换两种模式，TypeScript 编译通过，dev server 能跑。**

核心约束：
- 新代码放 `src/` 下的新文件，不动现有页面和 server.js
- 太阳系用原生 Three.js（非 R3F），组件通过 ref 和 React 通信
- 驾驶舱底图暂时用 CSS 占位（深棕 div + 简易窗框），等素材到位再替换
- 占位的视觉效果也要有氛围感——不是白板，是看起来"像 Outer Wilds 里的东西"
- 目录结构按 PRD.md 第五章的约定
